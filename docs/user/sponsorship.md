# 赞助权益体系

> 本规则规范 RelayPulse 通道赞助等级、权益与治理要求。规则可能根据运营情况调整，重大变更将提前通知。

## 1. 服务等级与权益

RelayPulse 按**通道**划分服务等级，分为两类：**自助等级**（自助流程，在线申请）和**商务等级**（人工对接，QQ:18058344）。赞助收入用于平台运营与实例可用性、质量等维度的监测。

所有等级均享有基础权益：浏览服务状态、查看历史数据、提交问题反馈、实时监测通知。

### 1.1 自助等级

适用场景：标准通道收录，通过站内自助流程完成。

| **监测频率** | 5m |

| **置顶资格** | — |

- 通过 [站内自助收录](/contact/apply) 申请，完成测试后提交，由平台审核后上线

>
> - `public`：难以核验通道是否真为公益性质
> - `signal`：个人赞助者极少

## 2. 入驻与收录

### 2.1 申请流程（自助等级）

适用于 `pulse` 等级的标准收录：

1. 前往 [自助收录页](https://relaypulse.top/contact/apply) 填写服务商信息，内联连通性测试验证端点可用性（**必选**，未通过不予受理）
2. 提交申请
3. 平台审核配置，如有问题通过 QQ（18058344）联调
4. 阅读并确认 [入驻须知与确认](sponsorship-agreement.md)
5. 通过 QQ（18058344）确认费用、发送 API Key 并完成付款
6. 配置验证通过后上线监测，权益即时生效

**不适合自助流程的场景**（请直接联系 QQ:18058344）：
- 需要 `beacon` 及以上商务等级
- 需要批量配置多个通道
- 特殊接入需求或非标配置

### 2.2 已收录通道变更

已完成收录的通道，如需修改通道信息，可通过 [站内变更申请](https://relaypulse.top/contact/change) 自助提交：

- 支持变更：服务商名称、官网链接、通道别名、分类
- 需通过测试验证：探测地址（`base_url`）、API Key 轮换

**不支持通过站内变更申请处理的事项**（请联系 QQ:18058344）：
- 赞助等级调整（升级/降级）
- 赞助续费
- 商务等级的通道配置事务

### 2.3 通道命名规范

所有通道的 `channel` 字段须遵循统一命名格式：`<渠道类型>-<来源简写>`

| 标识 | 含义 | 示例 |
|:---:|------|------|
| **O** | 官方通道 (Official) | `O-API`、`O-Web`、`O-AWS`、`O-GCP` |
| **R** | 逆向通道 (Reverse) | `R-自定义名` |
| **M** | 混合通道 (Mixed，默认) | `M-自定义名` |

**官方来源简写**：Web（网页订阅）、API（官方直连）、AWS（Bedrock）、GCP（Vertex AI）、App（合作应用）

> **过渡期**：现有未规范的通道名将在公告后 30 天内由平台统一迁移。过渡期内新旧名称并存，到期后以新名称为准。

### 2.4 付费方式

预付费制，按通道按月计费。具体支付方式以平台沟通确认为准。

- 到期未续费自动降级为 `pulse`（脉冲链路），不退还已付费用
- 因入驻方违约、授权瑕疵或合规风险核验成立而下架/终止的，不予退款
- 因平台自身经营调整等非入驻方原因提前终止的，按未履行期间比例退款

### 2.5 API Key 授权

入驻需同意平台将所提供的 API Key 基于监测目的使用，包括但不限于：
- **可用性监测**：定期发起 API 请求，检测连通性与响应延迟
- **深度质量检测**：通过各种技术手段检测通道的服务质量

API Key 仅用于配置验证、可用性监测、质量检测及与监测结果相关的必要展示，不用于与监测无关的其他用途。不同意授权则无法入驻。详见 [入驻须知与确认](sponsorship-agreement.md)。

## 3. 赞助升级与续费

### 3.1 升级申请

赞助等级升级由平台人工审核处理，升级申请通过 QQ（18058344）提交：

| 目标等级 | 参考条件 | 申请方式 |
|---------|------|---------|
| pulse | — | 站内自助申请 |
| beacon / backbone / core | 位于主板、可用率达标、收录一定时长 | QQ:18058344 人工对接 |

> **说明**：商务等级的升级条件（收录时长、可用率等）为人工审核时的参考因素，不由系统自动校验，以双方确认为准。

### 3.2 到期与续费

通道配置 `expires_at` 字段（格式 `YYYY-MM-DD`）标记赞助到期日。到期日当天仍有效，次日起自动降级为 `pulse`（脉冲链路）并移入备板（secondary board）。到期前续费并更新 `expires_at` 可保持原等级与权益不变。

续费事宜请通过 QQ（18058344）处理。

## 4. 质量治理

### 4.1 板块自动调整

系统每 30 分钟评估通道的 **7 天滚动可用率**，执行三级自动移板：

- **主板 → 备板**：可用率低于 **50%** 时自动移入备板
- **备板 → 主板**：可用率恢复至 **55%** 以上时自动回到主板
- **自动冷板**：可用率低于 **10%** 时自动移入冷板并**停止探测**（sticky，不会自动恢复）
- 评估需最少 **10 次**有效探测数据，数据不足时不做判断
- 自动移板仅影响板块归属，**不影响赞助等级和其他权益**

> 主板 ↔ 备板之间 5% 的缓冲带用于防止频繁切换。自动冷板为单向 sticky 状态：一旦触发，需管理员在配置中设置 `auto_cold_exempt: true` 手动解除，通道才会恢复探测并重新参与可用率评估。若管理员同时保留 `auto_cold_exempt: true`，该通道在恢复探测后即使可用率再次低于阈值也不会被再次自动冷板。

### 4.2 置顶规则

适用于 `beacon` 及以上等级，需同时满足：无负向标签（negative annotation）、可用率不低于平台配置阈值（具体数值以部署配置为准）。

**排序规则：**
1. 赞助级别降序：core > backbone > beacon
2. 同级别按可用率降序
3. 同可用率按响应延迟升序（低延迟优先）

**置顶限制：**
- 全局置顶数量受 `sponsor_pin.max_pinned` 限制（默认 10）
- 超出限额时按排序规则截断，未入选者按普通排序展示

**置顶时机：**
- 仅页面初始加载时生效，用户手动排序后以用户选择为准

**多模型组与置顶：**
- 多模型监测组（parent/child）的置顶资格由父通道元数据决定（sponsor_level、annotations），子通道不独立参与置顶判定
- 同一 `provider + service` 组合最多占用一个置顶名额
- 不支持以子通道作为主展示行进行置顶（reverse grouping）：子通道不携带独立赞助元数据，反转展示会导致去重键冲突和配额规则歧义

### 4.3 模型质量一致性

- 入驻方须保证通道实际提供的模型质量与所声明的渠道一致；如声明为官方渠道，则须确保实际来源为官方 API。
- 如发现实际提供的模型质量与所声明的渠道不一致，平台可视情节自主决定，可以但不限于采取加负向标签、下架（`hidden`）、移入冷板（`cold board`）等措施。

## 5. 风险声明与条款

### 5.1 治理与反馈

- **反馈渠道**：[GitHub Issues](https://github.com/prehisle/relay-pulse/issues)
- **处理流程**：反馈提交 → 平台受理 → 调查核实 → 结果通知 → 申诉机制。平台将尽快处理，不承诺具体时限。
- **负向标签**：仅针对经核实的严重问题（如持续失联、服务长期中断、多用户投诉等），展示至问题解决，之后可申请撤销。适用于 `pulse` 及以上商业通道；历史 `public` 通道亦适用，`signal` 不适用。
- **义务**：各等级需接受用户反馈并配合问题调查，平台负责中立协调。

### 5.2 平台承诺

1. **数据真实性**：保证不篡改监测数据
2. **中立协调**：反馈处理中保持中立，提供沟通支持
3. **规则透明**：规则变更提前公示通知
4. **隐私保护**：在采取合理安全措施的前提下保护赞助方敏感信息，不向无关第三方披露

### 5.3 风险提示与免责

1. **数据滞后性**：监测数据反映既往技术状态，不构成对未来可用性的保证。
2. **服务定位**：RelayPulse 仅提供技术监测服务，不提供交易撮合、资金托管或担保服务。
3. **赞助说明**：赞助标识仅代表对应等级的展示与监测权益，不构成对服务商信誉或履约能力的背书。赞助置顶仅代表展示优先级，不构成品质排名。
4. **责任边界**：用户与第三方服务商之间的交易风险由双方自行承担。平台不对第三方服务的质量、履约或资金安全承担担保责任。
5. **资金安全提示**：中转 API 行业存在经营风险，建议遵循「用多少充多少」原则，避免大额预充值。高可用率仅代表技术稳定性，不代表商家信誉。
6. **合规声明**：平台不对入驻方的上游授权来源、备案资质或经营合规性做担保。入驻方应自行确保其服务符合适用法律法规。

### 5.4 规则变更与协议确认

- 规则变更将通过网站公告、GitHub Issue 或 QQ 通知，重大变更自下一计费周期或续费时生效；涉及合规、安全等必要变更可即时生效。既有权益保持至赞助到期。
- 入驻前请仔细阅读本规则及 [入驻须知与确认](sponsorship-agreement.md)，完成入驻流程即表示您已充分理解并接受上述条款。
