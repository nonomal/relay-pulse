# 入驻须知与确认

> 本须知系入驻合作规则的组成部分，与 [赞助权益体系](sponsorship.md) 共同构成双方合作约定。

本须知适用于所有申请入驻 RelayPulse 平台的通道服务商（以下简称「入驻方」）。入驻前请仔细阅读，完成入驻流程即表示您已充分理解并接受以下条款。

---

## 1. API Key 授权与安全

- 入驻方提供的 API Key 仅用于配置验证、可用性监测、质量检测及与监测结果相关的必要展示，不用于与监测无关的其他用途。
- 平台通过环境变量或加密配置方式存储 API Key，不会在公开渠道明文展示。
- 如发生安全事件导致 API Key 可能泄露，平台将尽快通知入驻方并配合更换。
- 合作终止后，平台将在合理期限内删除入驻方的 API Key（法律法规另有要求或为处理争议而需要保留的除外）。

## 2. 入驻方陈述与保证

入驻方确认并保证：

- 对所提供的 API 端点、API Key、品牌名称及相关链接拥有合法使用权。
- 其服务运营符合适用法律法规及上游平台的服务条款。
- 提交的信息真实准确，如有变更应及时通知平台。
- 所申请收录的通道实际提供的模型质量与所声明的渠道一致；如不一致，平台有权采取包括但不限于加注负向标签、下架、移入冷板等措施。
- 如因入驻方自身的合规问题导致平台受到第三方追究，入驻方应配合处理并承担相应责任。

## 3. 免责与风险声明

- **非信誉背书**：赞助标识（标签、置顶等）仅代表展示与监测权益，不构成平台对入驻方商业信誉、资金安全或服务质量的背书或推荐。赞助置顶仅代表展示优先级，不构成品质排名。
- **合规免责**：平台不对入驻方的上游授权来源、备案资质或经营合规性做担保。入驻方应自行确保其服务符合适用法律法规。
- **交易风险自担**：用户与入驻方之间的交易关系由双方自行建立和维护，平台不参与交易过程，不承担担保责任。
- **数据局限性**：监测数据反映特定时间、特定探测方式与特定网络环境下的技术观测，存在固有局限性。详见 [监测方法论说明](methodology.md)。
- **资金安全提示**：中转 API 行业存在经营风险，平台建议用户遵循小额充值原则。高可用率仅代表技术稳定性，不代表商家信誉。

## 4. 下架与终止

- **平台风控下架**：平台保留因合规风险、严重用户投诉或不可抗力等原因，单方暂停或终止入驻方展示的权利。
- **通知义务**：非紧急情况下，平台将提前 3 个工作日通知入驻方；涉及紧急合规风险时可立即执行。
- **到期降级**：赞助到期未续费的，自到期次日起自动降级为 `pulse`（脉冲链路），不退还已付费用。
- **退款规则**：
  - 因入驻方违约、授权瑕疵或合规风险核验成立而下架/终止的，不予退款。
  - 因平台自身经营调整等非入驻方原因提前终止的，按未履行期间比例退款。

## 5. 争议解决

- 因本须知引发的争议，双方应首先友好协商解决。
- 协商不成的，任一方可向有管辖权的争议解决机构寻求救济。

## 6. 版本与变更

- 本须知随赞助规则主文档更新，重大变更经通知后自下一计费周期或续费时生效。
- 涉及合规、安全等必要变更可即时生效。
- 有效通知方式包括：网站公告、GitHub Issue 或 QQ。
